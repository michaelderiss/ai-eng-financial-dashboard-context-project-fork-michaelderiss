# Rule 05 - Governance and Documentation Maintenance

## Scope
Applies whenever architecture, dependencies, behavior, or contributor workflow changes.

## Rule
Any meaningful repository change must include maintenance updates to governance and operational context artifacts.

## Required checks
1. Keep rules in .agents/rules actionable and repository-specific.
2. Keep memory-bank artifacts up to date when introduced in Phase 4.
3. Avoid generic process notes that are not tied to observable code behavior.

## Why this exists
This project challenge explicitly evaluates stewardship quality and long-term maintainability artifacts.
